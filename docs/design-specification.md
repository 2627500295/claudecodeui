# 设计规格说明书 - 模型选择器和思维模式增强功能

## 1. 设计概述

### 1.1 设计目标
本设计旨在为Claude Code UI提供直观、高效的模型和思维模式选择界面，通过视觉化的控件让用户能够精确控制AI的行为特性，同时保持整体界面的简洁性和一致性。

### 1.2 设计原则
- **直观性**: 通过颜色编码和视觉指示器让用户快速理解当前状态
- **一致性**: 与现有UI组件保持视觉和交互的一致性
- **效率性**: 最小化用户操作步骤，提供快速切换能力
- **可访问性**: 支持键盘导航和屏幕阅读器
- **响应性**: 适配不同屏幕尺寸和设备类型

## 2. 视觉设计

### 2.1 整体布局设计

#### 2.1.1 布局结构
```
┌─────────────────────────────────────────────────────┐
│                   聊天消息区域                         │
├─────────────────────────────────────────────────────┤
│  [Model] [ThinkMode] [PermMode] [ScrollBtn]        │
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │  @ 文件引用   [📎] 输入框内容  [🎤] [➤]  │   │
│  └─────────────────────────────────────────────┘   │
│           提示文本和快捷键说明                      │
└─────────────────────────────────────────────────────┘
```

#### 2.1.2 组件位置规划
- **垂直位置**: 位于聊天输入框正上方，与输入框区域保持3个单位间距
- **水平布局**: 左对齐排列，从左到右：模型选择器 → 思维模式选择器 → 权限模式选择器
- **空间分配**: 使用flex布局，组件间3个单位的gap间距

### 2.2 模型选择器设计

#### 2.2.1 视觉外观
```css
基础样式:
- 尺寸: px-3 py-1.5 (水平12px，垂直6px内边距)
- 形状: rounded-lg (8px圆角)
- 字体: text-sm font-medium (14px中等字重)
- 边框: border (1px边框)
- 过渡: transition-all duration-200 (200ms过渡效果)
```


### 2.3 思维模式选择器设计

#### 2.3.1 视觉外观
基础样式与模型选择器保持一致，但内部结构包含强度指示器。

#### 2.3.2 强度指示器设计
```
思维强度可视化：
┌─────────────────────────────────┐
│  ▌▌▌▌▌ Auto                   │  ← 5个渐进高度的条形指示器
└─────────────────────────────────┘

条形规格：
- 宽度: w-1 (4px)
- 高度: h-2, h-3, h-3.5, h-4, h-5 (8px, 12px, 14px, 16px, 20px)
- 间距: gap-0.5 (2px)
- 形状: rounded-full (完全圆角)
```

#### 2.3.3 颜色状态映射
| 思维模式 | 主色调 | 激活条数 | 非激活色 |
|----------|--------|----------|----------|
| Auto | 灰色系 | 1条 | bg-gray-300 |
| Think | 蓝色系 | 2条 | bg-gray-300 |
| Think Hard | 青色系 | 3条 | bg-gray-300 |
| Think Harder | 靛蓝系 | 4条 | bg-gray-300 |
| Ultrathink | 紫色系 | 5条 | bg-gray-300 |

### 2.4 交互状态设计

#### 2.4.1 鼠标悬停状态
```css
hover样式增强:
- 背景色加深: hover:bg-{color}-100
- 暗色模式: dark:hover:bg-{color}-900/30
- 光标: cursor-pointer
- 过渡动画: transition-all duration-200
```

#### 2.4.2 点击反馈
- 无额外点击状态，直接切换到下一选项
- 切换过程中保持视觉连续性
- 新状态立即生效并反映在UI上

#### 2.4.3 焦点状态
```css
focus样式:
- 轮廓: focus:outline-none
- 环形高亮: focus:ring-2 focus:ring-{color}-500
- 偏移: focus:ring-offset-2
```

## 3. 响应式设计

### 3.1 断点策略
```css
断点定义:
- 移动设备: < 640px (sm)
- 平板设备: 640px - 768px (md) 
- 桌面设备: > 768px (lg+)
```

### 3.2 移动端适配

#### 3.2.1 尺寸调整
- 选择器按钮保持最小44px触摸目标
- 文字大小在小屏幕上保持可读性
- 间距在移动端适当缩小

#### 3.2.2 布局调整
```css
移动端布局:
- 保持水平排列，但可能换行
- 优先显示最重要的控件
- 必要时可隐藏部分装饰元素
```

### 3.3 暗色主题适配

#### 3.3.1 颜色映射
```css
暗色主题调整:
- 背景色: dark:bg-{color}-900/20
- 文字色: dark:text-{color}-300  
- 边框色: dark:border-{color}-600
- 悬停色: dark:hover:bg-{color}-900/30
```

## 4. 动效设计

### 4.1 过渡动画
```css
基础过渡:
- 属性: transition-all
- 时长: duration-200 (200ms)
- 缓动: ease-in-out (默认)
```

### 4.2 状态切换动画
- 颜色变化平滑过渡
- 指示器状态变化渐进显示
- 避免突兀的视觉跳跃

### 4.3 微交互设计
- 悬停时轻微背景色变化
- 点击时无明显延迟
- 状态变化立即可见

## 5. 可访问性设计

### 5.1 语义化标记
```html
无障碍属性:
- title: 提供操作说明
- role: button (隐式)
- aria-label: 描述当前状态
```

### 5.2 键盘导航
- Tab键在选择器间导航
- Enter/Space键激活选择器
- 焦点状态清晰可见

### 5.3 屏幕阅读器支持
- 提供有意义的标签文本
- 状态变化会被读屏软件感知
- 颜色信息不作为唯一信息载体

## 6. 组件架构设计

### 6.1 数据结构设计
```typescript
interface ModelOption {
  value: string | null;
  label: string;
}

interface ThinkingModeOption {
  value: string | null;
  label: string;
}

interface SelectorProps {
  currentValue: string | null;
  options: ModelOption[] | ThinkingModeOption[];
  onChange: (value: string | null) => void;
  colorScheme: ColorScheme;
}
```

### 6.2 状态管理设计
```javascript
状态管理策略:
- 本地组件状态: useState
- 计算属性缓存: useMemo  
- 回调函数优化: useCallback
- 渲染优化: React.memo
```

### 6.3 组件组合设计
```
SelectorGroup
├── ModelSelector
│   ├── SelectorButton
│   └── ColorIndicator
├── ThinkingModeSelector
│   ├── SelectorButton
│   ├── IntensityIndicator
│   └── ColorIndicator

```
